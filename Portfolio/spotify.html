Overview

Spotify is a commercial music streaming, podcast and video service that provides millions of digital rights management-restricted media to users worldwide. In a two-week allotted time window, I created a basic web-based artist discovery application leveraging Spotify's API and Materialize CSS, a CSS framework based on Google's Material design.

Role

Designer, Front-End Developer

  • HTML
  • SCSS
  • JavaScript
  • JQuery
  • AJAX
  • Materialize CSS
  • RWD

Process

Before diving into the development of this assignment, I had to not only envision what the interface would look like on both desktop and mobile devices utilizing Materialize CSS, but also determine what components of the Spotify catalog, or API endpoints, would be relevant for the purposes of artist discovery.

Once a general layout had been determined for both desktop and mobile devices (Image 1. Initial sketches shown right/above.), I explored the various API endpoints made available through Spotify and decided to include the following: (1) the originally searched artist's most popular track; (2) the ability to follow the originally searched artist; (3) the originally searched artist's top four related artists and (4) their respective most popular tracks; and (5) the ability to play each of these tracks directly from the website application (provided the user is logged into his/her own Spotify account).

Image 2. Pre-expanded primary artist card. Tablet view.

Image 3. Pre-expanded and expanded secondary artists cards. Tablet view.

Development

The final piece is a completely responsive web application that includes all aforementioned features built on the Materialize CSS framework.
